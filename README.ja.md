# 検査仕様 打合せサイト について

本社・台湾支社・仕入先（現在は「銘環」様のみ対象）が、検査仕様に関する資料を確認し、
ダウンロードし、コメントでやり取りできるサイトです。ログインが必要で、本社・台湾・銘環それぞれに
共有のアカウントを発行しています（詳細は「3. ログインについて」）。

**サイト URL**：https://jameswei-afk.github.io/ikankensa/

---

## 1. 主な機能

### 一覧ページ
- 対象品目（メーカー＝「銘環」、かつ「銘環確認」欄に具体的な内容がある品目）を一覧表示
- 顧客・品番・品名で検索可能

### 品目詳細ページ
- 顧客／メーカー／品番／品名を表示
- 関連資料をダウンロード可能：
  - 購入仕様書／購入仕様図／製品外観目視検査基準／検査基準書／検査成績書
  - 同じ種類の資料が PDF と Word／Excel の両方存在する場合は、PDF のみを表示（重複を避けるため）
- コメント機能：
  - コメントする立場を「日本早川／台湾早川／銘環」の3択から選択（自由入力ではない）
  - 画像またはファイルを1つ添付可能（5MB まで、画像／PDF／Word／Excel。スクリーンショットを直接貼り付けも可）
  - 新しいコメントはリアルタイムで反映（再読み込み不要）
  - 自分が投稿したコメントは自分のブラウザからのみ削除可能（ログイン不要の仕組み）
- 日本語／中文の表示切り替えボタン（右上）
- 閲覧にはログインが必要（詳細は次項）

## 2. データを最新版に更新する方法

本社・台湾で毎週の打ち合わせを終え、Excel（`20250821_購入仕様書_完了状況一覧.xlsx`）の内容を確定したら、
担当者（現在は魏さん）のPCで以下のコマンドを実行するだけで、サイトの内容が自動的に最新化されます。

```
cd C:\Users\q5695\Documents\meikan-vendor-portal
python publish.py
```

このコマンドは以下を自動で行います：
1. Excel を読み直し、対象条件（メーカー＝銘環、銘環確認が空欄でない）に合う品目を抽出
2. 各品目に対応するフォルダを探し、関連資料ファイルを取得
3. 新しく条件を満たした品目をサイトに追加
4. 既存品目のファイルが更新されていれば、サイト側のファイルも自動更新
5. 変更がない品目・ファイルはそのまま（何度実行しても安全＝再実行可能）

**サイトへの反映は実行後すぐです**（画面デザインの変更とは違い、承認待ちや反映待ちの時間はありません）。

## 3. ログインについて

サイトを開くと最初にログイン画面が表示されます。本社・台湾・銘環それぞれに **共有アカウント（1組ずつ）**
を発行しており、各社内で同じアカウントを使い回していただく形です（個人ごとのアカウントではありません）。

- ログイン情報（メールアドレス／パスワード）は、このドキュメントとは別の安全な手段で共有します
- 一度ログインすれば、次回以降は同じブラウザなら自動的にログイン状態が保たれます
- ログアウトしたい場合は、画面右上の「ログアウト」ボタンから

アカウントの追加・削除・パスワード再設定は、担当者（魏さん）が Supabase の管理画面から行います。

## 4. 議論が完了した品目を削除する方法

品目の検討が完了し、サイトに載せておく必要がなくなったら、以下のコマンドで完全に削除できます
（品目データ・関連資料ファイル・コメント・コメントの添付ファイルをすべて一括削除）。

```
python remove_item.py PS-00046           # まずプレビュー（何も削除されない）
python remove_item.py PS-00046 --yes     # 内容を確認後、実際に削除
```

⚠️ **この削除は元に戻せません**。実行前に必ずプレビュー（`--yes` を付けない状態）で品番が正しいか確認してください。
また、Excel 側の「銘環確認」欄がまだ入力されたままだと、次回 `publish.py` 実行時にサイトへ再度追加されてしまうため、
削除する際は Excel 側の該当欄も空欄または「-」に変更してください。

## 5. セキュリティについて

本サイトは社内・仕入先間の**簡易的な打ち合わせツール**であり、機密情報を扱う基幹システムと同等の
セキュリティレベルは想定していません。採用している対策と、認識しておくべき制約は以下の通りです。

### 実施している対策
- **ログイン制限**：ログインしないと、品目一覧・詳細・コメントなど、あらゆるデータの閲覧ができない。
  データベース（Supabase）側のアクセス制御（RLS）でも、ログインしていないリクエストは拒否される
  （画面上のログイン画面を回避してデータベースに直接アクセスを試みても、同様に拒否される）
- **通信は常に暗号化**（サイト・データベースともに HTTPS／TLS）
- **データの書き込み制限**：ログイン済みユーザーでもコメントの投稿以外のデータは書き換えられない
  （品目情報や資料ファイルは管理者の秘密鍵を持つ `publish.py` からしか更新できない）
- **コメント削除の保護**：コメントごとにランダムな削除用キーを発行し、投稿した本人のブラウザにしか
  保存しないため、他人のコメントは削除できない仕組みになっている
- **管理者用の秘密鍵（service_role key）**は魏さんの PC 内の `.env` ファイルにのみ保存し、
  GitHub 等の公開の場所には一切アップロードしていない
- ソースコード（GitHub）は公開リポジトリだが、機密情報・パスワード類は含まれていない

### 認識しておくべき制約
- ログインは**本社／台湾／銘環それぞれ1組の共有アカウント**であり、個人単位のアカウントではない。
  そのため「誰が」ログインしたかまでは特定できない
- ログインすれば、**一覧ページから対象品目すべて**（現状は「銘環」向けの品目全件）を閲覧できる。
  特定の品目だけを見せる、といった限定共有はできない
- 資料ファイルのダウンロード用リンク自体は、一度取得されて外部にコピー・転送されると、
  ログインしていない状態でもそのリンクからアクセスできてしまう（リンクの推測は困難だが、
  リンクが漏れた場合に無効化する仕組みはない）
- コメント投稿に対するスパム防止（reCAPTCHA 等）は導入していない。不適切な投稿があった場合は
  管理画面（Supabase）から手動で削除する運用になる
- 誰が・いつ閲覧したかのアクセスログは記録していない
- データは自社サーバーではなく、外部クラウドサービス（Supabase／GitHub Pages、共に無料プラン）上に
  保管されている

### 総括
ログイン制限の導入により、リンクを知っているだけの第三者が資料を閲覧することは防げるようになりました。
一方で、共有アカウントであるためユーザー単位の特定はできない点、ダウンロードリンク自体は転送されると
ログインなしでも開ける点は、引き続き認識しておく必要があります。
**厳格な機密管理が必要な情報の共有には、これでも十分ではありません**。対象は現状「銘環」向けの
購入仕様関連資料に限定しており、**期間限定の打ち合わせツール**として、台湾検査移管に伴う打ち合わせが
完了した時点でシステムごと削除・停止する想定です。
